Design Tokens

Semantic tokens from the Design System. All tokens live in tokens.css as CSS custom properties on :root. Link before Tailwind CDN.

Source: DESIGN_SYSTEM.md

Token Groups

PrefixPurpose

--fg-*Foreground elements — icons, UI chrome
--bg-*Background surfaces and overlays
--border-*Border colors
--button-*Button variant colors
--input-*Form input colors
--nav-*Sidebar navigation states
--tab-*Tab navigation states
--status-*Badge/status indicator colors

Rules

  • Always use semantic tokens. Never use hardcoded hex values in components.
  • Never use raw palette tokens. Add a new semantic token rather than using hex directly.
  • Token intent matters. Use tokens for their described purpose:
    • --bg-surface → white card/panel backgrounds
    • --bg-subtle → secondary surface backgrounds, metric cards, monospace pills
    • --bg-page → page-level background
    • --bg-accent → accent-colored backgrounds, historical changelog borders
    • --bg-warning → warning/alert banner backgrounds (amber)
    • --nav-* → sidebar nav only
    • --tab-* → tab navigation only
    • --status-* → badge and status indicators only
    • --status-active-* → "Active" campaign/IO badges specifically (distinct green from success)

Color Notes

Clarifications for similar or easily confused tokens.

  • Active vs Success greens. --status-active-* and --status-success-* are different greens. Use --status-active-* only for "Active" campaign/IO badges. Use --status-success-* for complete, confirmed states. Never substitute one for the other.
  • Warning backgrounds. --bg-warning (#fbbf24) is amber for full-width warning/alert banners (e.g. High Burn Rate). --status-warning-bg (#fffbeb) is light yellow for small badge pills. They are not interchangeable.
  • Links. Use --text-link (#4F46E5) for links and external value references in pills.
  • Accent backgrounds. --bg-accent (#EEF2FF) is for accent-colored surfaces and historical changelog borders.
  • Foreground elements. --fg-primary and --fg-secondary are for icons and UI chrome, not body text.
