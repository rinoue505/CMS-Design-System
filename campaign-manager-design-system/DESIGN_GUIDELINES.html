Component Patterns

Sidebar

Width 230px. Sections: Logo area → Create row (+ Create pill + chevrons) → Navigation (gap 8px) → Spacer → User profile. Nav active: bg-[#f1f5f9], font-medium. Inactive: font-normal, text-[--color/slate/600].

Tab Bar

px-[32px] py-[16px], border-b. Tabs: gap 4px, each px-[10px] py-[8px] rounded-[8px]. Active: border #d9d9d9. Settings dropdown: bg-[--bg/subtle], right-aligned.

Data Table Rows

Header: py-12px. Data: py-20px (Segments) or py-24px (Campaigns), px-32px. Hierarchy: child rows pl-[48px] + 2px indigo bar. Row actions: 4×16px ellipsis on far right.

Status Badges

rounded-[8px] px-[8px] py-[2px]. Active: bg #d2f0df, text #11552f. Inactive: bg slate/200, text slate/600. Condition badges: bg slate/100, text 13px.

Form Inputs

Label: font-medium 14px, text-secondary. Required: red asterisk. Input: border slate/300, rounded-8px, px-16px py-12px.

Action Bar (Bottom)

Fixed bottom, border-t. px-24px py-16px, justify-between. Left: Cancel (text #595959). Right: Step indicator + Primary button (bg overlay, white text, rounded-6px).

Progress Bar

Track: bg-subtle, h-6px or h-12px, rounded-12px/16px. Fill: fg-primary or fg-secondary or red for over-budget.

Icon Sizes

12px (small inline), 14px (nav), 16px (alert, chevron), 20px (active nav), 24px (sidebar chevrons), 32px (avatar).

Implementation Notes for Coding Agents

  • Convert Tailwind to project stack. Map Figma MCP output to your design system (CSS variables, theme, styled-components).
  • Preserve exact values. Use specified px for spacing, font sizes, colors. Avoid arbitrary rounding.
  • Reuse shared components. Sidebar, NavItem, TabBar, Badges, form fields appear across screens. Extract them.
  • Hierarchy in tables. Parent/child rows use pl-48px and 2px vertical bar. Maintain for segment/campaign hierarchies.
  • Responsive behavior. Design uses fixed widths. Consider breakpoints (e.g. collapsible sidebar).
  • Asset URLs. Figma MCP asset URLs expire after 7 days. Download and commit icons, or use icon library.
